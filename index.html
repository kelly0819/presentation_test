장기 취향 / 단기 취향

PREFERENCE MODEL
장기 취향 (Long-term Preference)

변화가 느린 고정 성향

퍼스널 컬러, 피부 타입, 민감성 여부 등 장기적으로 잘 변화하지 않는 요소를 0~1 정규화하여 사용자 성향 벡터로 활용

활용: 협업 필터링 과정에서 코사인 유사도 기반 사용자 간 성향 비교에 사용

단기 취향 (Short-term Preference)

시간에 따라 변화하는 선호

계절감, 트렌드, 최근 관심사, 구매 목적 등은 시간에 따라 변화하는 사용자 선호 정보

📝 컨텍스트 누적 저장

사용자의 최근 행동 및 검색 기록과 함께 컨텍스트 형태로 누적 저장

예시: "여름용 가벼운 쿠션 필요" · "면접용 자연스러운 메이크업" 등 구매 사유 텍스트를 임베딩하여 Vector DB에 저장

INPUT · CACHE
Step 1 — 입력 방식
📡

NFC 태깅

태그 인식 → URL query parameter 자동 전달

이미지 업로드

직접 촬영 또는 캡처본 → VLM 분석

⌨️

텍스트 입력

상품명 / 브랜드명 직접 입력

NFC 플랫폼 고려 사항

Android

Web NFC API 지원 → 웹에서 태그 스캔 시 자동 분석 가능

iOS

Safari/WebKit에서 Web NFC 미지원 → React Native/Flutter 앱 기반 구현 필요

Step 2 — Redis 캐시 전략
캐시 키: 상품명 + 브랜드 + 옵션 조합
HIT → 기존 분석 결과 즉시 반환  |  MISS → Agent 파이프라인 실행 후 결과 저장

캐시 정책

자주 조회되는 상품의 분석 결과를 Redis에 TTL (Time To Live) 기반으로 저장 (ex. 2주)

동일 상품에 대한 중복 분석 및 LLM 호출 비용 감소 목적

// Cache Flow
key = "${productName}:${brand}:${option}"

if (redis.get(key)) {
  return cachedResult  // HIT → 즉시 반환
} else {
  result = runAgentPipeline()
  redis.set(key, result, "EX", 1209600)
  return result  // MISS → 저장 후 반환
}

Agent 3 — 개인화 추천 (Hybrid Architecture)

RECOMMENDATION ENGINE
① 협업 필터링

'사용자 벡터'끼리 비교 → 코사인 유사도 계산
유사도 높은 사용자 Top-N
→ 그 사용자들이 선호한 상품 후보 Top-N 추출
Python → DB 조회 → NumPy / scikit-learn

② 개인화 추천 (Content-based)

장기 취향 정보를 추천 보정값으로 활용
• 건성 피부 → drying 감점
• 민감성 → irritative 성분 감점
• 가성비 → 고가 상품 감점

③ 구매 사유 RAG

구매 사유 텍스트 임베딩 → Vector DB 저장 → 유사 사유 검색 → LLM이 선호 보정
현재 입력과 유사한 구매 사유를 RAG 기반으로 검색

최종 추천 = 장기 취향 + 상품 Feature + Context(RAG)
# 암묵적(행동) × 0.4 + 명시적(RAG) × 0.6
final_vec[f] = existing[f] * 0.4 + rag[f] * 0.6

# 예시
long_lasting: 0.7×0.4 + 0.9×0.6 = 0.82
drying: 0.3×0.4 + 0.1×0.6 = 0.18 ↓ 건조함 회피 강화
Cold Start 시: 협업 필터링 생략 → RAG + 올리브영 카테고리 상위 아이템 추천으로 보완
RAG 결합 이유
이유설명
암묵+명시 결합무엇을 선택했는지 + 왜 선택했는지 동시 반영
정확도 향상건조함·트러블 등 회피 feature 정밀 반영
Cold Start 완화소량 텍스트만으로 초기 추천 품질 보완
설명 가능성"지속력이 중요한 사용자" 등 추천 근거 제시
